# 系統規格書：【SwipeBooks】即時競技 Web App

## 1. 專案概述 (Project Overview)
本專案為一款結合 Tinder 滑卡機制與 Kahoot 競技計分的會計學互動教學平台。系統具備「模組化副本 (Dungeons)」設計，老師可依據當日課程進度選擇不同的題庫副本，並透過大螢幕管理遊戲大廳（QR Code 加入、人數監控、重啟遊戲）。學生端則透過左右滑動卡片進行限時競技作答。

## 2. 技術架構建議 (Tech Stack)
* **前端框架：** React.js (Next.js) + Tailwind CSS。
* **動畫套件：** `framer-motion` (處理卡片 Swipe 物理特效與排行榜 Bar Chart Race)。
* **後端與資料庫：** Firebase Realtime Database (處理即時狀態廣播與大廳人數統計)。
* **部署：** GitHub Actions 自動部署至 Zeabur。

## 3. 資料庫結構更新 (Firebase RTDB Schema)
加入副本庫與大廳控制狀態：

| 節點 (Node) | 結構與欄位說明 | 功能描述 |
| :--- | :--- | :--- |
| `/dungeons/{dungeonId}` | `title`, `description`, `questions` (Array of objects) | 系統預設的「副本題庫」。 |
| `/rooms/{roomId}` | `status` (lobby/playing/podium), `selected_dungeon`, `current_q_index`, `is_randomized` (bool) | 控管教室狀態。老師可隨時將 status 改回 lobby 來「重啟遊戲」。 |
| `/rooms/{roomId}/questions` | Array of ordered question IDs | 當老師點擊「開始遊戲」時，將選定副本的題目複製過來並進行**陣列洗牌 (Shuffle)**，確保每次順序不同。 |
| `/rooms/{roomId}/players` | `{ playerId: { name, score, is_active } }` | 即時統計已連線加入的學生。 |

## 4. 老師端規格：大螢幕戰情中心 (Host Dashboard)
老師端介面分為三個主要階段，擁有絕對的房間控制權：

### 階段一：等待大廳 (The Lobby)
* **UI 佈局：** * 畫面左側：顯示巨大的 **動態 QR Code** 與 6 碼數字房間代碼（例：9527-88）。
    * 畫面右側：動態彈出已加入學生的姓名泡泡。
    * 畫面頂部：顯示醒目的 **「目前加入人數：X 人」** 計數器。
* **控制面板 (Control Panel)：** * 提供下拉選單讓老師選擇「今日挑戰副本」（如：買賣業會計）。
    * 點擊「開始遊戲 (Start Game)」按鈕 $\rightarrow$ 觸發後端隨機洗牌題目陣列，並跳轉至發題畫面。

### 階段二：戰鬥發題 (Battle Arena)
* **UI 佈局：** 中央顯示目前題目與巨大倒數計時器；下方顯示已作答人數進度條。
* **控制面板：** 提供「強制跳下一題 (Skip)」按鈕。

### 階段三：戰況結算 (Leaderboard & Restart)
* **UI 佈局：** 顯示前 5 名總積分排行榜，與本題的左右滑動分佈圓餅圖。
* **控制面板：** * 提供「下一題 (Next Question)」按鈕。
    * 提供紅色警戒按鈕 **「重新啟動遊戲 (Reset Room)」** $\rightarrow$ 點擊後清空所有玩家分數，保留玩家名單，狀態切回 Lobby，準備挑戰下一個副本。

## 5. 學生端規格：手機電競駕駛艙 (Player App)
* **加入流程：** 掃描大廳 QR Code 後，輸入暱稱即進入等待畫面（顯示：「等待老闆發題中...」）。
* **操作機制：** 看到題目卡牌後，向左滑代表「借 (Debit)」，向右滑代表「貸 (Credit)」。
* **防呆機制：** 網路斷線重連時，依賴本地 `localStorage` 的 `playerId` 自動加回大廳。

## 6. 初始內建副本劇本庫 (Dungeon Content Seeders)

開發時需預先載入以下 3 個副本資料，供老師隨機抽考：

### ⚔️ 副本一：【新手村】正常餘額直覺挑戰 (Normal Balances)
* **教學目標：** 訓練學生對五大會計要素增加時，應該記在借方還是貸方的反射神經。
* **題目清單 (向左滑 👈借 / 貸👉 向右滑)：**
    * 現金增加 (答案：👈 借)
    * 應付帳款增加 (答案：👉 貸)
    * 銷貨收入發生 (答案：👉 貸)
    * 薪資費用發生 (答案：👈 借)
    * 股本增加 (答案：👉 貸)
    * 預付租金增加 (答案：👈 借)

### ⚔️ 副本二：【進貨大亨】買賣業會計循環 (Merchandising)
* **教學目標：** 熟悉永續盤存制下的進銷貨交易與折讓。
* **題目清單 (向左滑 👈借 / 貸👉 向右滑)：**
    * 向廠商賒購商品，請問「存貨」要放哪？ (答案：👈 借)
    * 承上題，請問「應付帳款」要放哪？ (答案：👉 貸)
    * 顧客來店用現金買衣服，請問「銷貨收入」要放哪？ (答案：👉 貸)
    * 承上題（永續盤存制），請問「銷貨成本」要放哪？ (答案：👈 借)
    * 承上題，賣出衣服的「存貨」要放哪？ (答案：👉 貸)
    * 發現進貨商品有瑕疵，退回給廠商，請問「存貨」要放哪？ (答案：👉 貸)

### ⚔️ 副本三：【期末審判】虛實帳戶大清洗 (Closing Entries)
* **教學目標：** 結帳時，將虛帳戶結清歸零的反射訓練。
* **玩法變異提示：** 畫面的左右提示改為「👈 借方結清 / 貸方結清 👉」。
* **題目清單 (要想辦法把這個科目的餘額變不見)：**
    * 年底要把「銷貨收入」結清，請問結帳分錄中「銷貨收入」要放哪？ (答案：👈 借。因原為貸方餘額)
    * 年底要把「租金費用」結清，請問「租金費用」要放哪？ (答案：👉 貸。因原為借方餘額)
    * 年底要把「銷貨退回」結清，請問「銷貨退回」要放哪？ (答案：👉 貸。因原為借方餘額)
    * (陷阱題) 年底要把「預收收入」結清，請問放哪？ (答案：不用滑/時間到扣零分，因為它是負債，是實帳戶不能結清！)
